# cybersecurity-notes

Module 1: Introduction to Cyber Security
🔹 What is Cyber Security?

Protecting:

Systems
Networks
Data

From:
Unauthorized access
Attacks
Damage

🔹 Types of Cyber Security

Network Security → protects networks
Application Security → protects software
Information Security → protects data
Operational Security → policies & procedures

🔹 CIA Triad (VERY IMPORTANT)

Confidentiality → only authorized access
Integrity → data is not changed
Availability → data is accessible when needed

🔹 Threat vs Vulnerability vs Risk

Threat → possible danger (hacker, malware)
Vulnerability → weakness (bug, weak password)
Risk → chance of damage

🔹 Types of Hackers

White Hat → ethical hacker
Black Hat → malicious hacker
Grey Hat → in between

Module 2: Networking Fundamentals

🔹 What is a Network?

Connection of devices to share data

🔹 Types of Network

LAN → Local Area Network
WAN → Wide Area Network (Internet)

🔹 IP Address

Unique identifier of a device

Example:
192.168.1.1

🔹 Types of IP

Public IP → internet visible
Private IP → internal network

🔹 DNS (Domain Name System)

Converts domain → IP

Example:
google.com → 142.250.x.x

🔹 Common Ports (VERY IMPORTANT)

21 → FTP
22 → SSH
23 → Telnet
25 → SMTP
53 → DNS
80 → HTTP
443 → HTTPS

🔹 Protocols

Rules for communication

Examples:
HTTP / HTTPS → web
FTP → file transfer
SSH → secure remote login

🔹 OSI Model (Basic idea)

7 layers:

Physical
Data Link
Network
Transport
Session
Presentation
Application

👉 Just remember flow (don’t stress too much now)

Module 3: How The Web Works

🔹 What happens when you visit a website?

Enter URL
DNS converts to IP
Browser sends request
Server responds
Page loads

🔹 HTTP vs HTTPS

HTTP → not secure
HTTPS → encrypted (secure)

🔹 Request & Response

Request:
Browser → Server

Response:
Server → Browser

🔹 Status Codes

200 → OK
301 → Redirect
403 → Forbidden
404 → Not Found
500 → Server Error

🔹 URL Structure

https://example.com/page
https → protocol
example.com → domain
/page → path

🔹 Cookies

Small data stored in browser
Used for login/session

🔹 Basic Web Concepts

Frontend → what you see
Backend → server side
Database → stores data
